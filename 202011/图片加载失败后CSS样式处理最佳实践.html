# 图片加载失败后CSS样式处理最佳实践

转载：张鑫旭 博客

一、传统实现

图片的异常处理是工作中经常遇到的问题，有时候因为网络或资源路径等问题，加载失败时<img>标签呈现的并不友好，视觉效果太过简陋；于是设计师通常用一张占位图来代替默认加载异常的情况，触发使用占位图可以通过onerror事件，代码如下：

<img src="xxx.jpg" alt="图片描述" onerror="this.src=error.jpg">
1

配合css object-fit属性，实现替换内容缩放适应容器的宽高比：

img[src$="error.jpg"] {
    object-fit: contain;  // cover | fit | scale-down
}
1
2
3

object-fit属性值含义：如下

contain

  • 被替换的内容将被缩放，以在填充元素的内容框时保持其宽高比。 整个对象在填充盒子的同时保留其长宽比，因此如果宽高比与框的宽高比不匹配，该对象将被添加“黑边”。

cover

  • 被替换的内容在保持其宽高比的同时填充元素的整个内容框。如果对象的宽高比与内容框不相匹配，该对象将被剪裁以适应内容框。

fill

  • 被替换的内容正好填充元素的内容框。整个对象将完全填充此框。如果对象的宽高比与内容框不相匹配，那么该对象将被拉伸以适应内容框。

none

  • 被替换的内容将保持其原有的尺寸。

scale-down

  • 内容的尺寸与 none 或 contain 中的一个相同，取决于它们两个之间谁得到的对象尺寸会更小一些。

然而以上这种实现方式有一个不好的问题，原生图片出错会展示alt信息，用占位图替代后展示信息没有了，因此需要进一步优化

二、最佳实现，同事显示alt信息

为了解决alt信息显示和维护困难的问题，张鑫旭博客中提到了一种方案，那就是图片加载失败onerror不再替换src地址，而是增加一个错误类名，例如：

<img src="xxx.jpg" alt="图片描述" onerror="this.classlist.add('error');">
1

再配合css样式修改（以下只是博客中的审美，具体待定）

img.error {
  display: inline-block;
  transform: scale(1);
  content: '';
  color: transparent;
}
img.error::before {
  content: '';
  position: absolute;
  left: 0; top: 0;
  width: 100%; height: 100%;
  background: #f5f5f5 url(break.svg) no-repeat center / 50% 50%;
}
img.error::after {
  content: attr(alt);
  position: absolute;
  left: 0; bottom: 0;
  width: 100%;
  line-height: 2;
  background-color: rgba(0,0,0,.5);
  color: white;
  font-size: 12px;
  text-align: center;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

三、个人的一点补充

其实懒加载也是非常不错的一种解决方案，比如vue-lazy或原生lazyload属性,相比上面的方案懒加载用法更简单，但是灵活性不如这种方案，毕竟样式是随时可以修改的
