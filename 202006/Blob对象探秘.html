# Blob对象探秘

Blob( Binary Large Object ) 二进制类型的大对象，通常是影像/声音或多媒体文件

一、Blob是什么

为了更直观的感受 Blob 对象，我们先来使用 Blob 构造函数，创建一个 blob 对象，具体如下所示：

var blob = new Blob(['我'],{type:'text/plain'})

console.log(blob) // Blob {size: 3, type: "text/plain"}
1
2
3

​ 如你所见，blob包含两个属性 :size和type, 其中 size 属性用于表示数据的大小（以字节为单位），type 是 MIME 类型的字符串 。至于一个中文为什么是三个字节，其实不同操作系统下中文占的字节数是变化的，这里不用管他

二、Blob API 简介

  • 构造函数

    var aBlob = new Blob(blobParts, options);

相关参数说明如下：

  • blobParts: 是一个由ArrayBuffer , ArrayBufferView , Blob ,DOMString等对象构成的数组.DOMString会被编码为UTF-8.

  • options : 一个可选的对象，包含以下两个属性

    type --默认值为“ ” ，它代表放到blob中的数组内容的MIME类型

    endings -- 默认值为 “ transparent ” , 用于指定包含行结束符 \n 的字符串如何被写入, 代表会保持 blob 中保存的结束符不变 。还有一个可选值：“native” , 代表行结束符会被更改为适合宿主操作系统文件系统的换行符

    tips : MIME（Multipurpose Internet Mail Extensions）多用途互联网邮件扩展类型 ,是设定某种扩展名的文件用一种应用程序来打开的方式类型，当该扩展名文件被访问的时候，浏览器会自动使用指定应用程序来打开。多用于指定一些客户端自定义的文件名，以及一些媒体文件打开方式。

    常见的 MIME 类型有：超文本标记语言文本 .html text/html、PNG图像 .png image/png、普通文本 .txt text/plain 等。

示例一：字符串创建

var blobParts = ["<html><h1>hello world</h1></html>"]
var blob = new Blob(blobParts,{type:'text/html',endings:'transparent'})

var hello = new Uint8Array([72, 101, 108, 108, 111]); //二机制格式 “hello”

  • 属性

    前面我们已经知道Blob对象的两个属性：

    size( 只读 ) ：数据大小 ，字节为单位

    type(只读) ：数据类型，默认为 “ ”

  • 方法

    slice([start[, end[, contentType]]])：返回一个新的 Blob 对象，包含了源 Blob 对象中指定范围内的数据

    stream(): 返回一个能读取blob内容的ReadableStream

    text() : 返回一个Promise对象且包含blob所有内容的utf-8格式的USVString

    arrayBuffer(): 返回一个Promise对象且包含blob所有内容的二进制格式的ArrayBuffer

    注意：blob对象是不可改变。不能直接在原对象中更改数据，但是可以对其进行分割 ，再创建新的对象。这种行为类似于字符串

三、 使用场景

    1. 分片上传

    File对象是特殊类型的Blob, 且可以用在任意的Blob类型的上下文中。所以针对大文件传输的场景 ，我们可以使用slice方法对文件进行切割，然后分片进行上传，具体示例如下：

    const file = new File(["a".repeat(1000000)], "test.txt");

const chunkSize = 50000;
const url = "https://httpbin.org/post";

async function chunkedUpload() {
  

  for (let start = 0; start < file.size; start += chunkSize) {
      const chunk = file.slice(start, start + chunkSize + 1);
      const fd = new FormData();
      fd.append("data", chunk);

      let result = await fetch(url, { method: "post", body: fd }).then((res) =>res.text());
      console.log(result)  //发送20次请求 打印20个结果
  }
}

    1. 用作URL

    浏览器内部每次调用 URL.createObjectURL(blob) 都会生成一个映射一个唯一的源，形式为blob:<origin>/<uuid>，对应示例如下：

    blob:http://127.0.0.1:5500/5d873c7a-a5f9-4911-bedb-b2f604b13fc9

    这个url看似很不错，但实际上它也有副作用。会导致blob对象也常驻内存，浏览器无法释放它，只有等到文档卸载。如果应用的寿命很长，势必会造成内存的浪费，即使不再需要blob对象 它也会在内存中。

    针对这个问题 ，我们可以调用 URL.revokeObjectURL(url) 方法 从内部映射中删除引用，从而允许删除 Blob（如果没有其他引用），并释放内存。接下来，我们来看一下 Blob 文件下载的具体示例。

    <button id="downloadBtn">文件下载</button>

//js
const download = (fileName, blob) => {
  const link = document.createElement("a");
  link.href = URL.createObjectURL(blob);
  link.download = fileName;
  link.click();
  link.remove();
  URL.revokeObjectURL(link.href);
};

const downloadBtn = document.querySelector("#downloadBtn");
downloadBtn.addEventListener("click", (event) => {
  const fileName = "blob.txt";
  const myBlob = new Blob(["一文彻底掌握 Blob Web API"], { type: "text/plain" });
  download(fileName, myBlob);
});

    1. 转换为Base64

如果想把一个文件持久化存储为二进制数据（小文件推荐），以字符串的形式保存或传输，以下是将blob转换为base64编码并实现本地预览的示例：

<input type="file" accept="image/*" onchange="loadFile(event)">
<img id="output"/>

<script>
  const loadFile = function(event) {
    const reader = new FileReader();
    let file = event.target.files[0]
    reader.onload = function(){
      const output = document.querySelector('output');
      output.src = reader.result;
    };
    reader.readAsDataURL(file); //不明白这段代码为什么要放在onload下边，虽然都一样
  };
</script>

    1. 图片压缩

      我们希望上传图片时 先对图片进行一定的压缩，然后再提交服务器，从而减少传输消耗。前端要实现图片压缩，可以利用Canvas提供的toDataURL（）方法，两个可选参数，代码如下：

      // compress.js

const MAX_WIDTH = 800; // 图片最大宽度

function compress(base64, quality, mimeType) {  //图片压缩函数
  let canvas = document.createElement("canvas");
  let img = document.createElement("img");
  img.crossOrigin = "anonymous";
  return new Promise((resolve, reject) => {
    img.src = base64;
    img.onload = () => {
      let targetWidth, targetHeight;
      if (img.width > MAX_WIDTH) {
        targetWidth = MAX_WIDTH;
        targetHeight = (img.height * MAX_WIDTH) / img.width; //按比例减小
      } else {
        targetWidth = img.width;
        targetHeight = img.height;
      }
      canvas.width = targetWidth;
      canvas.height = targetHeight;
      let ctx = canvas.getContext("2d");
      ctx.clearRect(0, 0, targetWidth, targetHeight); // 清除画布
      ctx.drawImage(img, 0, 0, canvas.width, canvas.height);
      let imageData = canvas.toDataURL(mimeType, quality / 100);
      resolve(imageData);
    };
  });
}

//为进一步减少数据大小，再转换为Blob对象
function dataUrlToBlob(base64, mimeType) {
  let bytes = window.atob(base64.split(",")[1]);
  let ab = new ArrayBuffer(bytes.length);
  let ia = new Uint8Array(ab);
  for (let i = 0; i < bytes.length; i++) {
    ia[i] = bytes.charCodeAt(i);
  }
  return new Blob([ab], { type: mimeType });
}


//将Blob对象封装在FormData对象中  提交到服务器
function uploadFile(url, blob) {
  let formData = new FormData();
  let request = new XMLHttpRequest();
  formData.append("image", blob);
  request.open("POST", url, true);
  request.send(formData);
}

    1. 生成PDF文档

      PDF（便携式文件格式，Portable Document Format）是由 Adobe Systems 在 1993 年用于文件交换所发展出的文件格式。在浏览器端，利用一些现成的开源库，比如 jsPDF，我们也可以方便地生成 PDF 文档。

      <!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>客户端生成 PDF 示例</title>
  </head>
  <body>
    <h3>客户端生成 PDF 示例</h3>
    <script src="https://unpkg.com/jspdf@latest/dist/jspdf.min.js"></script>
    <script>
      (function generatePdf() {
        const doc = new jsPDF();
        doc.text("Hello semlinker!", 66, 88);
        const blob = new Blob([doc.output()], { type: "application/pdf" });
        blob.text().then((blobAsText) => {
          console.log(blobAsText);
        });
      })();
    </script>
  </body>
</html>
