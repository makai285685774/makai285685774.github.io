# 利用canvas 生成带有专属二维码的动态海报

需求：双十二活动期间，每一个参与活动的用户专门定制一张带有自己二维码的海报

思路整理

  1. 海报的背景图片是一张固定的大图
  2. 大图的固定区域描绘二维码
  3. 二维码由活动链接生成，扫描之后跳转

准备工作

海报最后合成一张大图，html5 中 canvas 自然避免不了；根据链接字符串生成二维码 ，比较之后 QRCode.js 是不错的选择 github地址

  2. 绘制背景海报，此时，如果一切正常的话，页面中就会出现一个二维码，先不去管他

    加载背景海报图

    上面是一张统一的背景图，因为此时不应该在页面看到它（没经过加工的半成品），所以display:none隐藏，此时还需要绘制后的容器 如下

    <img id="img3">
    1

    需要考虑的问题，浏览器窗口宽和高不固定，海报的宽和高是固定的，所以当窗口宽度大于海报宽度，绘制后的背景图片保持原图宽，而当窗口宽度小于海报宽度，绘制后的背景图应该等于窗口宽度， 至于高度还要看具体海报的实际高度，在不变形的前提下有所取舍，我的思路是宽和高都按百分比缩放（0.8），这样就可以保证图片所有部分都被绘制出来，并且按照原比例不变形

     var canvas = document.createElement("canvas"),  //画布=对象
     context = canvas.getContext("2d")  // 上下文=画笔
 var qrcodeBgImg = document.getElementById('qrcodeBgImg')  //背景图
 var ratio = 0.8;  //缩放比例
 
var CanvasBgWidth = $(document).width() >450?qrcodeBgImg.width:$(document).width()
var CanvasBgHeight = $(document).height() >800?qrcodeBgImg.height:$(document).height()

  $(canvas).css({  //画布大小
      'width': CanvasBgWidth * ratio,
      'height': CanvasBgHeight * ratio
  });

var wScale = CanvasBgWidth / 450

context.drawImage(codeImg, 175 * wScale * ratio, 595 * hScale * ratio, 100 * wScale * ratio, 100 * wScale *ratio);

需要注意，加载绘制好的图片属于异步操作，所以layer.open应该在加载图片的回调函数里执行，前四步封装的函数叫 draw() ,使用图片预加载代码如下

//点击之前二维码已经存在Dom中
$('.share').click(function (e) {  //点击按钮执行
	var img = new Image() 
    img.src=draw()  //赋值后img 对象开始加载图片
     
    img.onload=function(){  // 图片加载完成
        img3.src=img.src  //img3 容器，已存在Dom中，用于存放绘制完成的图片容器
        // 省略layer.open() //弹出img3图片层
    }
}

图片预加载说明，浏览器的缓存机制，同一个资源（例如图片）,只有第一次加载是异步耗时的（当然手动清理缓存除外），当再一次加载同一个资源，默认从缓存中取。预加载就是利用这个原理，预先在内存中加载资源，等到Dom中需要该资源时直接从缓存获取 ，使用户感知不到加载的过程

img3 为什么没有执行 var img3 =document.getElementById ('img3')` ,就可以直接使用?

这个源自于javascript 这门语言的特点 ， 任何在Dom中存在Id 的标签<div id='element'><div>, window对象中都存在一个 全局 的element对象，和 var element =document.getElementById ('element')` 效果是一样的
